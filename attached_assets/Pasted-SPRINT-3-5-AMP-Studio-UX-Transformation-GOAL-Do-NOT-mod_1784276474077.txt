SPRINT 3.5 — AMP Studio UX Transformation

GOAL

Do NOT modify business logic.

Do NOT modify models.

Do NOT modify permissions.

Do NOT remove Django Admin.

Instead, create a completely new experience for users after login.

=================================================

CURRENT PROBLEM

The current login redirects users directly into Django Admin.

This interface is suitable for developers but confusing for radio owners and non-technical users.

Users should interact with AMP Studio Dashboard first.

Django Admin becomes an advanced maintenance tool only.

=================================================

OBJECTIVES

Transform AMP Studio into the main application.

Django Admin must become secondary.
