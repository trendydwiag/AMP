
SPRINT 3.6 — PLATFORM UI CONSISTENCY & NAVIGATION HARDENING
IMPORTANT
Ini BUKAN sprint penambahan fitur.
Ini sprint QUALITY ASSURANCE.
Jangan mengubah business logic.
Jangan rollback pekerjaan Sprint 3.4 maupun Sprint 3.5.
Fokus hanya memperbaiki inkonsistensi seluruh AMP Studio.

OBJECTIVE
AMP Studio harus terasa sebagai SATU aplikasi profesional.
Semua halaman wajib menggunakan design system yang sama.
Tidak boleh ada halaman yang terasa dibuat oleh komponen berbeda.

TASK 1
Audit seluruh halaman Studio
Scan seluruh route:
/studio/ /studio/streaming/ /studio/program/ /studio/episode/ /studio/jadwal/ /studio/host/ /studio/content/ /studio/articles/ /studio/podcast/ /studio/community/ /studio/media/ /studio/analytics/ /studio/partner/ /studio/settings/ /studio/setup/
Cari seluruh halaman yang:
* memakai layout berbeda
* memakai sidebar berbeda
* memakai navbar berbeda
* memakai padding berbeda
* memakai warna berbeda
* memakai card berbeda
* memakai font berbeda
* memakai spacing berbeda
Samakan semuanya.
Gunakan SATU layout global.

TASK 2
Gunakan Base Layout Tunggal
Seluruh halaman Studio HARUS menggunakan:
base_studio.html
Tidak boleh ada halaman yang masih memakai:
dashboard.html
legacy_admin.html
old_layout.html
atau layout lama lainnya.
Semua harus extend base layout yang sama.

Contoh:

Episode
Jadwal
dst.
Tetap sama di seluruh aplikasi.

TASK 7
Audit Semua Menu
Klik satu-persatu seluruh menu sidebar.
Pastikan:
tidak ada 404
tidak ada template error
tidak ada page kosong
tidak ada placeholder
tidak ada TODO
tidak ada halaman legacy
tidak ada route mati
Jika menu belum memiliki fungsi:
buat halaman minimal yang layak.

TASK 8
Bersihkan Comment yang Bocor
Saat ini ditemukan comment seperti
{# radio-player.js must load BEFORE homepage.js ... #}
dan
{# Sprint 3.5 P3: Radio Health Widget ... #}
yang tampil ke user.
Cari seluruh file.
Hilangkan seluruh template comment yang ikut dirender.
Pastikan tidak ada lagi internal developer notes yang muncul di UI.

Definition of Done
Sprint dianggap selesai jika:
* Tidak ada halaman dengan layout berbeda.
* Tidak ada menu yang mati.
* Tidak ada komentar developer tampil di UI.
* Semua halaman memakai base layout yang sama.
* Semua menu mengarah ke tujuan yang benar.
* Seluruh dashboard terasa seperti satu produk profesional.
* Tidak ada perubahan business logic, hanya peningkatan kualitas UI/UX, konsistensi, dan stabilitas platform.
