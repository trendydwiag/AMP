====================================================
OBJECTIVE
====================================================

Target sprint ini adalah membuat Kabulhaden CMS terasa seperti produk yang siap digunakan oleh pemilik stasiun radio.

Bukan sekadar aplikasi developer.

Setiap halaman harus dapat digunakan untuk demo tanpa kebingungan, error, placeholder, atau UI yang belum selesai.

====================================================
AUDIT SELURUH CMS
====================================================

Audit seluruh halaman CMS satu per satu.

Periksa:

• Dashboard
• Program
• Jadwal
• Episode
• Podcast
• Berita
• Media
• Sponsor
• Komunitas
• Streaming
• Settings
• Users
• Platform (khusus superadmin)

Untuk setiap halaman pastikan:

✓ tidak ada template comment yang tampil
✓ tidak ada placeholder
✓ tidak ada TODO
✓ tidak ada Coming Soon
✓ tidak ada Lorem Ipsum
✓ tidak ada tombol mati
✓ tidak ada menu mati
✓ tidak ada link mati
✓ tidak ada breadcrumb salah
✓ tidak ada redirect aneh
✓ tidak ada UI yang berbeda dari AMP Studio
✓ tidak ada halaman menggunakan layout lama
✓ dark mode konsisten
✓ mobile layout tetap rapi

====================================================
EMPTY STATES
====================================================

Semua halaman yang tidak memiliki data harus memiliki Empty State yang baik.

Harus memiliki:

• ilustrasi/icon
• penjelasan singkat
• CTA utama

Contoh:

Belum ada Program
→ Buat Program Pertama

Belum ada Podcast
→ Buat Podcast Pertama

Belum ada Sponsor
→ Tambahkan Sponsor
