# Checklist Partner Baru

## Tujuan

Dokumen ini berisi daftar periksa saat menerima Partner baru.

## Kapan Digunakan

- Saat Super Admin menerima Partner baru
- Saat memulai proses onboarding Partner

---

## Daftar Periksa

### Penerimaan Partner
- [ ] Data Partner diterima
- [ ] Kontrak ditandatangani
- [ ] Pembayaran diterima
- [ ] Persetujuan dari manajemen

### Pembuatan Akun
- [ ] Partner dibuat di sistem
- [ ] Tenant diaktifkan
- [ ] Akun admin Partner dibuat
- [ ] Email undangan dikirim

### Pengaturan Dasar
- [ ] Logo Partner diunggah
- [ ] Warna tema diatur
- [ ] Informasi perusahaan diisi
- [ ] Domain ditentukan

### Pengaturan Teknis
- [ ] Domain dikonfigurasi
- [ ] SSL diaktifkan
- [ ] SMTP dikonfigurasi
- [ ] Stream URL ditentukan

### Pembentukan Tim
- [ ] Daftar tim diterima
- [ ] Akun untuk setiap anggota tim dibuat
- [ ] Role ditetapkan untuk setiap anggota
- [ ] Email undangan dikirim ke semua anggota

### Konten Awal
- [ ] Daftar program diterima
- [ ] Jadwal awal ditentukan
- [ ] Artikel pertama disiapkan
- [ ] Media disiapkan

### Pengujian
- [ ] Login berhasil untuk semua akun
- [ ] Website bisa diakses
- [ ] Stream radio berjalan
- [ ] Email notifikasi berfungsi

### Go Live
- [ ] Semua checklist selesai
- [ ] Persetujuan dari Super Admin
- [ ] Partner diaktifkan
- [ ] Siaran pertama dimulai

---

## Tips

- Ikuti langkah-langkah secara berurutan
- Dokumentasikan semua pengaturan
- Komunikasikan secara rutin dengan Partner
- Minta masukan dari Partner
- Perbaiki masalah segera

## Catatan

- Proses biasanya memakan waktu 2-4 minggu
- Setiap Partner memiliki kebutuhan berbeda
- Super Admin siap membantu jika ada masalah

## Troubleshooting

| Masalah | Solusi |
|---------|--------|
| Email undangan tidak masuk | Kirim ulang atau hubungi admin |
| Logo gagal diunggah | Periksa ukuran dan format file |
| Domain tidak bisa diakses | Periksa pengaturan DNS |
| Stream tidak berjalan | Periksa RadioBoss dan koneksi |

## FAQ

**Q: Berapa lama proses onboarding?**
A: Biasanya 2-4 minggu, tergantung kesiapan tim.

**Q: Apakah bisa go live tanpa semua checklist selesai?**
A: Tidak. Semua item harus selesai untuk go live.

**Q: Bagaimana jika ada masalah teknis?**
A: Hubungi tim teknis untuk bantuan.

## Best Practice

2. **Dokumentasikan semua pengaturan**
3. **Komunikasikan secara rutin dengan Partner**
4. **Minta masukan dari Partner**
5. **Perbaiki masalah segera**
